Validator

Validation is a way to validate document. That being said, for example, if a column is declared to be an email, it should also have correct format.

DAB won’t automatically validate a document. You must execute it manually using dab.validateDoc() call.

DAB will try to validate document if:

  • Its schema collection (i.e: attributes) isn’t empty
  • Explicitly defined in validator options

dab.validateDoc() is a synchronous function. It’ll return an Error if it fails to validate and null otherwise. More on this please click here

Example:

...
dab.createCollection({
  name: 'users',
  attributes: {
    username: { 
      type: 'string', 
      validator: { 
        required: true, 
        isAlpha: true 
      }
    },
    name: { 
      type: 'string', 
      validator: { 
      	required: true,
      	isAlphanumeric: true
      }
    },
    gender: { 
      type: 'boolean'
    },
    birth_date: { 
      type: 'date', 
      validator: { 
        isBefore: ['2000-01-01', 'YYYY-MM-DD'] 
      }
    },
    email: { 
      type: 'string', 
      validator: {
      	required: true, 
        isEmail: true
      }
    }
    ...
  }
})
...

From the example above:

  • username can’t be left blank and must contain letters only
  • name is required and must contain letters and numbers only
  • gender can be empty (null). If it’s not empty, it must be a boolean value (1, true, 0, false)
  • birth_date can be left empty. If not, it must be before January 1st 2000 and use ‘YYYY-MM-DD’ format
  • email is required and must be in correct format

All validator.js method are supported:

Validator Description
contains: "<seed>" Check if value contains the seed
equals: "<comparison>" Check if value matches the comparison
isAfter: true or isAfter: [<dt>, <format>] Check if value is a date that’s after now or after dt in moment.js specified format
isAlpha: true or isAlpha: "<locale>" Check if value contains only letters (a-zA-Z) in en-US or specific locale
isAlphanumeric: true or isAlphanumeric: "<locale>" Check if value contains only letters and numbers in en-US or specific locale
isAscii: true Check if value contains ASCII chars only
isBase64: true Check if value is base64 encoded
isBefore: true or isBefore: [<dt>, <format>] Check if value is a date that’s before now or before dt in moment.js specified format
isBoolean: true Check if a string is a boolean
isByteLength: {...} Check if value’s length (in UTF-8 bytes) falls in a range. More on options
isCreditCard: true Check if value is a credit card
isCurrency: true or isCurrency: {...} Check if value is a valid currency amount. More on options
isDataURI: true Check if value is a data uri format
isDecimal: true or isDecimal: {...} Check if value represents a decimal number. More on options
isDivisibleBy: <number> Check if value is a number that’s divisible by another
isEmail: true or isEmail: {...} Check if value is an email. More on options
isEmpty: true Check if value has a length of zero
isFQDN: true or isFQDN: {...} Check if value is a fully qualified domain name. More on options
isFloat: true or isFloat: {...} Check if value is a float. More on options
isFullWidth: true Check if value contains any full-width chars
isHalfWidth: true Check if value contains any half-width chars
isHash: "<algorithm>" Check if value is a hash of type algorithm. More on algorithm
isHexColor: true Check if value is a hexadecimal color
isHexadecimal: true Check if value is a hexadecimal number
isIP: true or isIP: "<version>" Check if value is an IP. version = 4 or 6
isISBN: true or isISBN: "<version>" Check if value is an ISBN. version = 10 or 13
isISSN: true or isISSN: {...} Check if value is an ISSN. More on options
isISIN: true Check if value is an ISIN (stock/security identifier)
isISO8601: true Check if value is a valid ISO 8601 date
isISO31661Alpha2: true Check if value is a valid ISO 3166-1 alpha-2 officially assigned country code
isISRC: true Check if value is a ISRC
isIn: [...] Check if value is in a array of allowed values
isInteger: true or isInteger: {...} Check if value is an integer. More on algorithm
isJSON: true Check if value is valid JSON
isLatLong: true Check if value is a valid latitude-longitude coordinate in the format lat,long or lat, long
isLength: {...} Check if value’s length falls in a range. More on algorithm
isLowercase: true Check if value is lowercase
isMACAddress: true Check if value is a MAC address
isMD5: true Check if value is a MD5 hash
isMimeType: true Check if value matches to a valid MIME type format
isMobilePhone: "<locale>" Check if value is a mobile phone number. More on locale
isMongoId: true Check if value is a valid hex-encoded representation of a MongoDB ObjectId
isMultibyte: true Check if value contains one or more multibyte chars
isNumeric: true Check if value contains only numbers
isPort: true Check if value is a valid port number
isPostalCode: "<locale>" Check if value is a postal code. More on locale
isSurrogatePair: true Check if value contains any surrogate pairs chars
isURL: true or isURL: {...} Check if value is an URL. More on options
isUUID: true or isUUID: "<version>" Check if value is a UUID. version = 3, 4, or 5
isUppercase: true Check if value is uppercase
isVariableWidth: true Check if value contains a mixture of full and half-width chars
isWhitelisted: "<chars>" Checks characters if they appear in the whitelist
matches: <pattern> or `matches: [, ] Check if string matches the pattern. More on pattern & modifier

Furthermore, basic validators and a few built-in validators that injected automatically by attribute types:

Validator Description
required: true Check if value is not empty
isDate: true or isDate: "<format>" Check if value is a date or value in moment.js specific format
isDatetime: true or isDatetime: "<format>" Check if value is a datetime or value in moment.js specific format
isObject: true Check if value is an object
isArray: true Check if value is an array
